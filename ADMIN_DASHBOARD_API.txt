ADMIN DASHBOARD & PACKAGE API
==============================
Date: April 19, 2026

NEW ENDPOINTS ADDED:
--------------------

1. GET /api/admin/dashboard
   - No auth required
   - Returns comprehensive dashboard with cards and metrics
   
   Response includes:
   - cards: Array of 9 metric cards with title, value, icon, color, description, percentage
     * Total Users
     * Active Users (with percentage)
     * Trial Users (with percentage)
     * Inactive Users (with percentage)
     * Total Payments
     * Approved Payments (with percentage)
     * Pending Payments (with percentage)
     * Monthly Revenue (approved payments this month)
     * Conversion Rate (trial to active %)
   
   - recent_activations: Last 10 approved payments with owner details
   
   - summary: Aggregated data
     * users: { total, active, inactive, trial }
     * payments: { total, approved, pending, rejected }
     * metrics: { conversion_rate, monthly_revenue }

2. GET /api/admin/dashboard/stats?period=month
   - No auth required
   - Query params: period (month, week, year)
   - Returns time-based statistics
   
   Response:
   {
     "period": "month",
     "start_date": "2026-03-31T18:15:00.000Z",
     "stats": {
       "new_users": 2,
       "new_payments": 2,
       "new_activations": 1
     }
   }

3. GET /api/packages
   - No auth required
   - Returns all packages with owner count
   
   Response:
   {
     "success": true,
     "packages": [
       {
         "package_id": "...",
         "package_key": "hardware",
         "package_name": "Hardware Store",
         "created_at": "...",
         "_count": { "owners": 15 }
       }
     ]
   }

4. GET /api/packages/:id
   - No auth required
   - Returns single package by ID with owner count

5. POST /api/packages
   - No auth required (admin endpoint)
   - Body: { "package_key": "string", "package_name": "string" }
   - Creates new package
   - Validates: case-insensitive duplicate check, trims whitespace
   - Example: "HARD" and "hard" are considered duplicates

6. PUT /api/packages/:id
   - No auth required (admin endpoint)
   - Body: { "package_key": "string", "package_name": "string" } (at least one required)
   - Updates package
   - Validates: case-insensitive duplicate check

7. DELETE /api/packages/:id
   - No auth required (admin endpoint)
   - Deletes package
   - Prevents deletion if package has owners


FEATURES:
---------

1. Service Layer Architecture:
   - Created packageService.js with business logic
   - Controller only handles HTTP requests/responses
   - Proper error handling with custom error codes and status codes

2. Duplicate Prevention:
   - Package keys are normalized (lowercase + trim)
   - Case-insensitive duplicate check
   - Clear error messages showing existing key

3. Dashboard Metrics:
   - Real-time user and payment counts
   - Conversion rate calculation
   - Monthly revenue tracking
   - Recent activations list
   - Percentage breakdowns for each metric

4. Flexible Stats:
   - Query by period (month, week, year)
   - New users, payments, and activations for selected period
